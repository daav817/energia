Project Overview:
I need to build a proprietary, containerized CRM and Brokerage Management web application for my energy brokerage firm, "Energia Power LLC". The application needs to run locally via Docker but must be designed for eventual cloud deployment.

Tech Stack:

Frontend: Next.js (App Router), Tailwind CSS, Shadcn UI (for clean, fast components).

Backend: Next.js API Routes (Node.js).

Database: PostgreSQL.

ORM: Prisma.

Core Architecture & Database Needs:
Please generate the Prisma schema.prisma model and the accompanying frontend/backend boilerplate for the following entities:

Customers: Must track basic CRM data. Crucially, must include boolean flags for has_electric and has_natural_gas.

Suppliers: Must track supplier details and contacts, also with is_electric and is_natural_gas flags.

Contracts: Linked to Customers and Suppliers. Must track energy type, start date, expiration date, and pricing.  There needs to be a way to track not only the price per energy type (i.e., $/MCF or $/CCF or $/KWH) but also a way to record contract income.  Ideally it would great if the system could either read emails from supplier which pertain to a monthly payment or access the supplier's portal to retrieve the latest monthly commission payment.

Tasks/Prospecting: A todo/note-taking system tied to either existing Customers or un-converted Prospects.

Documents: A table to store metadata and URLs (specifically Google Drive links) for attached bills and contracts.

Licenses: To track my CRNGS and CRES licenses. When a new license is entered with an "issue date", the system must automatically calculate and store an "expiration date" exactly 2 years out.
